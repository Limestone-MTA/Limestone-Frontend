File format in Limestone

To ensure accurate and efficient analysis, it is important that your data is properly organized before uploading it into Limestone.
This guide will help you understand the best practices for structuring your data so it can be easily processed, analyzed, and exported.

Use a Tabular Structure

Your data should be organized in a table format where:

  • Rows represent individual records (e.g., each row could represent a person, sample, or event).
  • Columns represent different variables or attributes for each record (e.g., age, salary, date, category).

Example:

Name Age Salary Hire Date
John Doe 28 50,000 2020-03-15
Jane Smith 34 60,000 2018-11-07
Sam Brown 41 75,000 2015-09-21

Use Clear and Consistent Column Headers

The first row of your dataset should contain column headers that clearly describe the variables in your table. Follow these tips:

  • Be descriptive: Use meaningful names (e.g., "Employee ID" instead of "ID").
  • Avoid special characters: Stick to letters, numbers, and underscores (e.g., "hire_date" instead of "Hire Date!").
  • Be consistent: Use uniform column names across datasets (e.g., always use "birth_date" instead of mixing "birth" and "dob").

Example of Clear Headers:

Employee ID Name Department Hire Date Salary
12345 John Doe Marketing 2020-03-15 50,000
67890 Jane Smith Finance 2018-11-07 60,000

Ensure Consistent Data Types

Each column should contain data of the same type to avoid errors. For example:

  • Text Fields: Use for names, categories, or descriptive data (e.g., "Department", "City").
  • Numerical Data: Use for quantities, amounts, or metrics (e.g., "Age", "Salary").
  • Dates: Use a consistent format for dates throughout the dataset (e.g., YYYY-MM-DD).

Handle Missing Data Appropriately

Missing data should be properly managed to ensure smooth uploads:

  • Leave cells blank for missing values rather than inserting "Unknown".
  • Or consider using standardized codes for missing data. Here is an exhaustive list of the missing codes we recognize : ['NA', 'Na', 'na', '#NA', '<NA>', 'N/A', 'n/a', '#N/A', 'NaN', '-NaN', 'nan', '-nan', '1.#IND', '-1.#IND', '1.#QNAN', '-1.#QNAN', 'NULL', 'null']

Example:

Name Age Salary Hire Date
John Doe 28 50,000 2020-03-15
Jane Smith 60,000 2018-11-07
Sam Brown 41 2015-09-21

Avoid Complex Structures

To avoid issues during data processing, ensure that:

  • There are no merged cells in your data.
  • Your data does not use multi-row headers.
  • Formulas are replaced with actual values, and special characters are avoided.

Use One Dataset or Sheet Per Upload

Ensure you're only uploading one dataset at a time. If your data is spread across multiple tables, combine them into a single dataset using a common identifier (e.g., "Employee ID" or "Product Code").
