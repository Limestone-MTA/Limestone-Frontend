Q&A Limestone

First, we would like to remind the users that this is a statistical analysis.
Choices at the quality control, analysis settings, and interpretation of results stages, should be done bearing in mind existing knowledge related to the data and to standard practices.
Performing analyses otherwise can result in invalid statistical tests.

Representation of missing values and true/false

Here is the list of missing value representation that is recognized by limestone :

  • [#N/A, #N/A, N/A, #NA, -1.#IND, -1.#QNAN, -NaN, -nan, 1.#IND, 1.#QNAN, <NA>, N/A, NA, NULL, NaN, None, n/a, nan, null, na, Na, ".."]

For the True/False please use these values :

  • [TRUE, True, true, YES, Yes, yes] for true values
  • [FALSE, False, false, NO, No, no] for false values
 Data types

The variables are categorized into 4 different types :

  • Text : This variable contains text element that are not readible as number. If the variables has less than 15 different values this variable is automatically transformed into 15 binary variables than can be analyzed.
  • Continuous : This variable is numerical with more than 15 different values.
  • Ordinal : This variable is numerical with less than 15 different values
  • Binary : This variable has 2 different values. It is represented with a 1 and a 0. The 1 is based on the first value in the variable column
 Sample size

If the sample size (i.e. the number of individuals, or entities) is less than (10 + number of variables in your model), your data can't be analyzed. With less than ~50 samples, you should be aware that results would be subject to major caution, and analyses warrant additional confirmation from an independent (and larger) dataset.

 Structured data

If your data are structured, and you do not account for it, this might invalidate the results from the analysis. Below are examples and potential solution.

  • If your data have a hierarchical structure (e.g. the variables in your dataset can be organized in a tree within a limited number of levels). We highly recommend that you analyze data from each level separately.
  • If the samples in your data are not independently and identically distributed (so called i.i.d.). This would be the case if, for example, your dataset includes multiple measurement of the same variable at multiple time points, or if you analyze individuals that are related. Analyzing such data requires advanced statistical techniques that are not currently implemented in Limestone. Therefore we currently recommend removing such sample or variables so that the remaining data are i.i.d.
 Genral information for quality control (QC)
  • Low count rule: Function applied to ordinal and binary variables only. When the occurrence of one value is very low (i.e. below the selected threshold), this value will be merged with the next smaller value. When it results in a constant (all samples have the same value), this variable is excluded.
  • Data transformation: Function that apply data transformation to all your continuous variables at once.
    • Log transformation will use the logarithm value of the variable instead of the original value.
    • Probit transformation will apply an inverse normal transformation on the ranked values, resulting in a fully normal distribution. Tie ranks are set with the average of the corresponding inverse of the percentile.
 Definition of variable type and use in the analysis setting (AS)
  • Outcomes refers to as dependent variable in a standard regression.
  • Predictors refers to as explanatory variable in a standard regression.
  • Confounders refers to variables that influences both the dependent variable and independent variable causing a spurious association. Variables checked as confounder will be systematically added as covariate in the analysis.
  • Covariates are variables that will be assessed by the MC algorithm. Those selected by MC (if any) will be included in the regression model as covariates.
